# PhDPal PhD Survive Guide

## Overview
Produce practical and emotionally grounded PhD survival guide content for PhDPal.
Keep advice actionable, realistic, and source-aware.

## Workflow
1. Collect inputs: target audience, platform, language, and source materials.
2. Read `references/sop.md` and follow the SOP sections in order.
3. Build one unit with `Scene`, `Pain`, `Laugh`, and `Do`.
4. Draft concise guidance with clear `problem -> action -> expected effect` logic.
5. Create an original visual concept (text card, checklist card, bingo card, or simple illustration).
6. Add caveats for uncertainty and avoid overstating claims.
7. Finalize post-ready output for the target platform.

## Output Contract
Return sections in this order:
1. `Post Draft`
2. `Visual Brief`
3. `Scene/Pain/Laugh/Do Mapping`
4. `Action Steps`
5. `Risks and Boundaries`
6. `Internal Sources Log`

## Rules
- Output in English unless the user explicitly requests another language.
- Do not fabricate claims, data, or citations.
- Separate evidence-based statements from interpretation.
- Keep tone grounded, empathetic, and practical; avoid over-dramatic emotional writing and avoid empty motivational language.
- Do not directly copy, translate line-by-line, or lightly rewrite external jokes.
- Do not reuse or trace external meme/comic images.
- Treat links in the resource pool as priority sources, not the only sources. Add better-fit references for PhD survival guidance when they are accurate and trustworthy.
- If evidence is weak, mark as tentative or remove the claim.
